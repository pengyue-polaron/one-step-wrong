# 产品开发总结

## 当前产品

`One Step Wrong` 现在包含两层互补体验：

1. 默认首页仍是三个可直接游玩的 NYU 数字安全章节。
2. `/studio` 是 OpenAI Build Week 的教师工作台，可以完成来源研究、人工审核、场景编译、有边界角色演练、确定性结局和证据复盘。

## 原有基础

- 「最后一次提交」深度桌面案例，包含网络、Brightspace、通知、账号响应与多结局。
- 「共享范围」与「这次是你吗」两个响应式决策章节。
- 统一案例注册表、通用决策引擎、独立深度模拟状态机与会话内进度。
- 因果链复盘、逐项止损、关键节点重玩和桌面/手机布局覆盖。

## Build Week 已实现

- 严格 `InstitutionProfile`：官方域名、来源记录、事实置信度、冲突、明确未知项、审核状态与品牌保护词。
- 可操作的人工审核：教师可以解决冲突事实、编辑来源支持的内容、逐项批准或拒绝来源，并查看研究警告与具体校验路径。
- 研究信任边界：显式官方域名不能被模型替换，来源必须使用 HTTPS，访问时间由服务器写入，精确品牌模式的授权确认保存在画像中。
- 审核过的 NYU 离线画像：Brightspace、Duo、Google Workspace、无线网络、钓鱼上报与学生报销材料均连接官方公开来源。
- 品牌安全编译：真实 NYU 画像进入 Scenario Architect，公开案例输出为虚构的 Northbridge University；受保护名称、域名或平台泄漏会导致验证失败。
- `The Voice You Know` 旗舰案例：普通报销任务、语音式冒名请求、三张有知识/频道边界的角色卡、证据、恢复步骤与四种确定性结局。
- 四条仅服务端 API：机构研究、场景生成、模拟回合和复盘。
- GPT-5.6 两阶段回合：Director 只选择允许事件；Role 只接收被选中角色的最小上下文并表演对话。
- 世界事实区分 `public` 与 `hidden`；事故真相不会混入 Director 公共事实或无关角色上下文。
- 纯确定性物理层：模型不能批准付款、共享文件、撤销访问、提交报告或选择结局。
- 声明式动作前置与受影响层恢复：未共享前不能撤销访问，只有实际受影响的层才进入遗漏恢复清单，Director 也只能建议当前可用动作。
- 学习者界面隐藏角色真实身份状态；教师仍可在生成包预览中检查角色边界。
- 审阅回退：无密钥、超时、越权事件、角色泄漏、异常结构或生成失败时仍可完整游玩。
- 规范化动作轨迹与复盘：安全路径不虚构恢复缺口，事故路径分别记录已完成和遗漏的恢复动作。
- 中英文 README、运行截图、`.env.example` 和 GitHub Actions CI。
- Next.js standalone 多阶段生产镜像；CI 会构建镜像并探测首页、Studio、fixture API 与非 root 运行用户。

## 当前证据

- `77` 条 schema、API、状态与组件测试。
- `13` 条 Playwright 浏览器测试。
- 浏览器覆盖 Studio 安全路径、事故恢复路径、自由文本回合、完整 replay 和 390 px 手机布局。
- 完整门槛：`lint`、`typecheck`、`test`、`build`、`test:e2e`。

## 尚需外部完成

以下项目不属于代码缺口，但在 `BUILD_WEEK_PLAN.md` 的最终 Definition of Done 中仍是必需证据：

- 配置有效 `OPENAI_API_KEY`，录制 GPT-5.6 在线研究、生成、Director、Role 与 Debrief 的真实调用证据。
- 将可使用离线 fixture 的版本部署到公开 URL。
- 按活动规则调整当前私有 GitHub 仓库的访问权限。
- 在主要 Codex 构建线程提交 `/feedback` 并记录 Session ID。
- 录制并公开不超过三分钟、带英文旁白或英文字幕的最终视频。

## 已知边界

- 不包含账号、数据库、跨会话进度、学校 SSO、LMS 集成或真实校园操作。
- 第一章多窗口桌面案例仍要求至少 1100 px；其余章节和 Studio 支持手机布局。
- 在线 GPT-5.6 路径需要网络和服务端密钥；离线评审路径不需要。
- 场景编译是有边界的声明式生成，不是通用爬虫、拖拽编辑器或无限 Agent 平台。
