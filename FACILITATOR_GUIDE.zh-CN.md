# 教师使用指南

<p>
  <a href="./FACILITATOR_GUIDE.md">English</a> ·
  <a href="./FACILITATOR_GUIDE.zh-CN.md"><strong>简体中文</strong></a>
</p>

本指南用于把 One Step Wrong 组织成一次简短的课堂、入学教育或团队学习活动。活动不需要账号、学习者追踪或 API key。

## 学习目标

学习者应当能够：

1. 区分熟悉感与已经验证的身份。
2. 解释一个验证渠道证明了什么、没有证明什么。
3. 在独立已知渠道确认之前，不执行高影响操作。
4. 区分遏制、恢复、沟通与上报步骤。
5. 把同一条判断规则用于不同任务。

## 活动形式

| 形式 | 时间 | 适用场景 |
| --- | ---: | --- |
| 个人演练 | 10 分钟 | 入学教育、咨询或自主学习 |
| 两人讨论 | 20 分钟 | 课堂、学生组织或员工工作坊 |
| 引导式对比 | 30–35 分钟 | 安全意识活动或教师带领的实验课 |

## 活动前

1. 打开 `/rehearsal` 进入旗舰演练，或打开 `/` 选择其他案例。
2. 自己先完成一次演练并打开教师报告。
3. 决定学习者单独操作、两人合作，还是共同观看投影。
4. 不要提前说明哪个动作最安全。只介绍普通任务，让时间压力自然发挥作用。
5. 不要要求学习者输入真实姓名、密码、付款信息或真实事件细节。

## 演练中

使用不会泄露答案的问题：

- 你现在真正需要完成的任务是什么？
- 这个渠道实际证明了什么？
- 哪些信息来自请求本身？
- 如果没有截止时间，你的判断会改变吗？
- 哪个动作会改变付款、访问权限或账号状态？

避免直接揭示答案：

- 哪个选项最安全？
- 你能看出这是骗局吗？
- 为什么这个请求是假的？

当学习者意见不一致时，让每个人说明支持动作的证据，而不是只对动作投票。

## 复盘与迁移

结果出现后：

1. 对照动作顺序，讨论每一步当时已经发现的证据。
2. 使用 Evidence Coach 对比一个由请求控制的渠道和一个独立已知渠道。
3. 在显示内置规则前，请学习者先用自己的话总结迁移规则。
4. 不回看原答案，直接完成新情境。
5. 打开教师报告，把首次演练结果和迁移结果放在一起讨论。

迁移动作比第一次结局更能说明学习效果。第一次走到安全结局可能来自既有知识或偶然选择；能在不同任务中重复使用规则，是更强的学习证据。

## 讨论问题

- 什么让便利动作显得合理？
- 哪条线索已经出现，却很容易被忽略？
- 熟悉的人名、品牌或语境是否替代了身份验证？
- 哪项检查真正离开了请求者控制的渠道？
- 如果不安全动作已经发生，应当先处理哪个受影响层？
- 这条规则如何用于文件共享、MFA、账号恢复或紧急文档请求？

## 轻量学习证据

One Step Wrong 不收集分析数据。小规模试用可以使用汇总或匿名证据：

- 统计有多少学习者在迁移情境中选择独立已知渠道。
- 使用一句匿名离场回答：“一个验证渠道更强，是因为……”
- 记录哪些证据区别引发了最多讨论。
- 只在群体层面对比首次演练和迁移结果，不记录个人身份。

不要把结局当成分数。真正有用的证据是学习者能否解释因果链，并在变化后的情境中重复使用判断规则。

## 可访问性与包容性

- 允许学习者按自己的速度阅读并重试关键节点。
- 动画影响舒适度或专注时，使用减少动态效果控制。
- 共同观看屏幕时，可以朗读角色消息和证据。
- 可以由一位伙伴操作界面，两人共同讨论。
- 不要求学习者披露过去经历过的真实安全事件。

## 教师检查清单

- [ ] 普通任务已经说清楚。
- [ ] 选择在结果出现前没有安全或危险标签。
- [ ] 学习者讨论的是证据，而不只是偏好的动作。
- [ ] 恢复过程按受影响层讨论。
- [ ] 已经完成新情境。
- [ ] 教师报告用于讨论，而不是个人评分。
- [ ] 没有记录真实凭据、私人事件或学习者身份。
