# 匿名试用工具包

本工具包用于进行小规模形成性试用，不会给 One Step Wrong 增加分析埋点、账号或学习者级记录。应用本身保持不变：教师只在活动结束后记录整场汇总计数，并在本地进行分析。

工具包可以展示一组学习者是否完成演练、迁移动作如何分布，以及多少解释符合预先定义的量规。它不能证明学习提升是由本产品造成的。

## 文件

- `templates/session-aggregate.csv` 是唯一接受的输入格式，只有表头，没有虚构结果。
- `MEASURES.md` 定义所有计数和解释量规。
- `scripts/analyze-pilot.mjs` 校验并汇总一场或多场活动。
- `pilot/local/` 已被 Git 忽略，用于保存本地工作数据。

## 开展一次活动

1. 按教师指南完成旗舰演练、复盘和新情境。
2. 给本场活动分配一个中性编号，例如 `S001`。编号中不要编码班级、学校、日期、教师或学习者身份。
3. 活动结束后，按照 `MEASURES.md` 的定义填写一行汇总计数。
4. 不要填写姓名、邮箱、电话号码、链接、人口统计、时间戳、原话、开放备注或真实事件描述。
5. 分析本地文件：

```bash
npm run pilot:analyze -- pilot/local/session-aggregate.csv
```

命令会输出 Markdown 汇总，不会输出活动编号或逐行记录。

## 隐私边界

分析器只接受：

- `S001` 格式的数字活动编号；
- 长度受限的场景版本；
- 一个受控的组织形式；
- 非负整数计数。

它会拒绝未知列、缺失列、带引号或自由文本的单元格、疑似个人信息、重复活动编号、负数、小数和不一致的合计。

如果学习者主动提到个人信息或真实事件，不要把这些内容写入试用文件。请遵守活动组织方关于知情同意、审核、保留和删除的规定。本工具包只用于形成性评估，不能替代经过批准的人体参与研究方案。

## 报告边界

只能把输出描述为探索性的群体层面证据。不要宣称：

- 产品造成了学习提升；
- 结果具有统计显著性；
- 产品对更广泛人群普遍有效；
- 可以判断个人能力；
- 未实际开展的活动产生了结果。

模板和分析器只是基础设施，并不是参与者证据。只有真实活动已经开展、汇总文件经过审核，并如实说明样本和过程后，才存在真实试用结果。
